# File review threads attach to file-level targets

Narview should sync GitHub review feedback that is anchored to an entire file as a **File Review Thread**. File Review Threads should attach to the relevant file-level **Review Target** when one exists, and remain visible in thread filters and **Human Feedback Packets** even when there is no specific line or symbol anchor.

Narview should also allow starting File Review Threads when file-level feedback is the correct GitHub anchor. Line-level Review Threads remain preferred for precise feedback, but full GitHub sync means users should not need to leave Narview to create whole-file review feedback.
